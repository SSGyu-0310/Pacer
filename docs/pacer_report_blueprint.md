# Pacer 포지션 리포트 — 재설계 블루프린트 v2

## 설계 원칙 3가지

1. **숫자 우선, 해설은 그 다음.** 환산점수·합격선·격차는 엔진이 계산해 화면에 먼저 박는다. AI는 이 숫자를 "왜 그런지" 풀어주는 역할만 한다.
2. **AI는 데이터 객체 안의 값만 서술한다.** AI에게 자유 생성 권한을 주지 않는다. 구조화된 JSON을 넘기고, 그 안의 필드(과목명·점수·격차·티어)만 가지고 문장을 쓰게 한다. → "영어 강점인데 강점 카드에 영어 없음", "과탐 안정 vs 탐구2 리스크 모순" 같은 일관성 붕괴가 원천 차단됨.
3. **제품이 해줄 일을 사용자 숙제로 넘기지 않는다.** "수학 전형을 탐색해보세요"가 아니라 "수학 50% 반영하는 △△·□□가 너와 맞는다"를 직접 보여준다.

이 원칙에 따라 각 섹션을 **[엔진]**(결정론적 계산) / **[AI]**(서술) / **[고정]**(템플릿 문구)으로 명확히 나눈다.

## 새 섹션 구조 (순서대로)

### 1. 헤더 — 포지션 한 줄 + 티어 배지
- **[엔진]** 티어 배지: `안정 / 적정 / 소신 / 상향` 중 하나. (영어 매치/리치를 쓰려면 `적정(Match)`처럼 한국어를 앞에 둔다.)
- **[AI]** 포지션 한 줄. 반드시 숫자에 앵커링.
  - ❌ "수학 강점과 탐구 변동성이 교차하는 출발점"
  - ✅ "연세대 △△ 작년 합격선 대비 +3.2점, 적정권에서 시즌을 시작합니다."

### 2. 핵심 숫자 패널 — **신규, 가장 중요**
지금 리포트에서 통째로 빠진 부분. 한눈에 들어오는 숫자 블록:

| 항목 | 값 |
|---|---|
| 내 환산점수 (전형 기준) | 712.4 |
| 작년 합격선 (70%컷) | 709.2 |
| **격차** | **+3.2** |

- 작은 게이지/막대로 내 위치를 시각화 (합격선=0 기준, 내 점수가 어디에).
- 컷 기준(70%컷/50%컷/최종등록자 등)은 라벨로 명시. 사용자가 무슨 숫자를 보는지 알아야 신뢰가 생긴다.
- **[엔진]** 전부 계산값. AI 개입 없음.

### 3. 지원 가능 라인 — **신규, Pacer의 핵심 가치**
실제 학과·전형 리스트. "연세대" 한 단어로 끝내지 않는다.

| 대학·학과 | 군 | 핵심 반영 | 내 점수 | 작년컷 | 격차 | 티어 |
|---|---|---|---|---|---|---|
| 연세대 △△학과 | 가 | 수학 40% | 712.4 | 708.1 | +4.3 | 적정 |
| 연세대 □□학부 | 나 | 수학 35%+탐구 30% | 705.0 | 709.8 | −4.8 | 소신 |
| ㅁㅁ대 ◇◇과 | 가 | 수학 50% | 715.2 | 706.0 | +9.2 | 안정 |

- 내 강점(수학)이 잘 먹히는 곳을 상단에 정렬.
- 각 행 클릭 시 상세(반영비 전체·역대 추이)로 연결.
- **[엔진]** 학과·반영비·컷·격차·티어 모두 계산. **[AI]** 리스트 위 1~2문장 요약만.

### 4. 강점 / 주의 — 기존 유지하되 전면 수정
**강점** (각 항목은 반드시 과목명+근거 숫자 포함)
- 수학: 백분위 98, 표준점수 상위권 → 수학 고반영 전형에서 +α
- 영어: 1등급 → 등급 감점 없음 (절대평가라 강점 카드에 **반드시 포함**, 빠지면 안 됨)
- (안정 과목이 있으면) 생명과학: 백분위 92 안정 유지

**주의** (과목 단위로 구체화, 서로 모순 금지)
- 탐구2(지구과학): 백분위 78, 변환표준점수 손해 구간 → 전형별 −2~4점 영향
- ※ "데이터 신뢰도 부족"은 여기서 **삭제** → 섹션 7 타임라인으로 이동(미래지향 프레이밍)

> **[AI] 프롬프트 제약 (필수):**
> - 강점/주의 항목은 입력 데이터의 `subjects[]` 배열에 있는 과목만 사용.
> - 한 과목을 강점과 주의에 동시에 넣지 말 것.
> - 모든 항목에 해당 과목의 숫자(백분위/등급/격차)를 1개 이상 포함.
> - 입력에 없는 점수·등급·학과를 새로 만들지 말 것.

### 5. What-if 시나리오 — **신규**
계산 엔진이 있으니 가능한 기능. Pacer만 할 수 있는 차별점.

- 탐구2를 한 등급 ↑ (78→백분위 88 가정) → 환산 **+2.6점** → 연세대 □□학부 **소신 → 적정** 진입
- 수학 표준점수 +3 → △△학과 격차 +4.3 → +7.0 (안정권)

- **[엔진]** 시나리오별 재계산. **[AI]** "어디를 올리면 무엇이 열리는지" 한 줄 해설.

### 6. 다음 행동 — 기존 유지, 단 위 숫자/시나리오에 연결
막연한 "취약 유형 보완" 금지. 시나리오와 묶는다.
- 지구과학 변환점수 손해 구간(II 단원 추정)을 우선 보완 → 시나리오상 +2.6점 효과
- 수학 50% 반영 ㅁㅁ대 ◇◇과를 안정 카드로 추가 검토
- 9월 모평 직후 이 리포트를 재생성해 격차 갱신

### 7. 타임라인 + 신뢰도 프레이밍 — **신규**
- `6월 ───●(지금) ─── 9월 ─── 수능` 축 위에 "지금 여기" 마커.
- 신뢰도 문구를 여기로 이동: "현재 6월 표본 기준이라 9월 모평 후 정밀도가 올라갑니다." (불안 조성이 아니라 추적 동기로)
- **[고정]** 템플릿.

### 8. 푸터 — CTA + 면책 (정리)
- **[고정]** 알림 신청 CTA 1개.
- **[고정]** 면책 문구 **1회만** (지금은 거의 동일한 게 2번 반복 → 1개로).
- `localhost` URL, `glm-5.1 · prompt v1` 메타데이터 → **프로덕션 비노출**. "AI 생성" 투명성 표기만 유지, 모델명/프롬프트 버전은 내부 디버깅용으로 숨김.

## 학생용 / 학부모용 차별화

토글이 장식이 되지 않으려면 정보의 **깊이와 목적**을 다르게 한다.

| | 학생용 | 학부모용 |
|---|---|---|
| 목적 | 동기부여 + 공부 방향 | 불안 해소 + 지금 도울 일 |
| 강조 섹션 | 3(라인) · 5(what-if) · 6(행동) | 1(티어) · 2(숫자) · 7(타임라인) |
| 톤 | "탐구2 올리면 □□가 열린다" | "결론부터: 적정권, 9월까지 탐구2가 변수" |
| 학과 상세 | 깊게 | 요약 (전공 나열보다 위험도) |
| 용어 | 백분위/변환점수 OK | 풀어서 설명 |

같은 데이터 객체 → 두 개의 프롬프트(섹션 강조·톤만 다름)로 분기.

## 엔진이 AI에 넘겨야 할 데이터 객체 (스키마 초안)

```json
{
  "student": {
    "subjects": [
      {"name": "수학", "percentile": 98, "strength": "high"},
      {"name": "영어", "grade": 1, "absolute": true, "strength": "high"},
      {"name": "생명과학", "percentile": 92, "strength": "stable"},
      {"name": "지구과학", "percentile": 78, "strength": "risk",
       "note": "변환표준점수 손해 구간"}
    ]
  },
  "myScore": 712.4,
  "scoreBasis": "연세대 △△ 환산식",
  "lines": [
    {"univ": "연세대", "dept": "△△학과", "group": "가",
     "keyWeight": "수학 40%", "cut": 708.1, "gap": 4.3, "tier": "적정"}
  ],
  "scenarios": [
    {"lever": "지구과학 78→88", "delta": 2.6,
     "unlocks": "연세대 □□학부 소신→적정"}
  ],
  "season": {"current": "6월", "next": "9월", "sampleConfidence": "low"}
}
```

AI는 이 객체 **밖의 값을 절대 만들지 않는다.** 모든 숫자·과목·학과는 여기서만 인용. 이게 일관성의 핵심 장치.

## 한 줄 요약
지금 리포트는 **AI 서술이 8할, 데이터가 0할**. 새 구조는 **데이터(엔진)가 뼈대, AI가 살**. 섹션 2·3·5(숫자·실제 라인·what-if)가 추가되고, AI는 자유 생성에서 "데이터 객체 해설"로 역할이 좁아진다.
