# 핵심대(CORE) 리스트 결정 + 검수 우선순위

검수 대상 "핵심대"를 어디까지로 보고, 무엇을 먼저 검수할지 합의한 문서다. 운영 소스는
[`packages/reference-data/data/review/core-universities.json`](../../packages/reference-data/data/review/core-universities.json)
이고, 이 문서는 그 **결정 배경과 작업 우선순위**를 사람이 읽기 위한 것이다. 데이터 상태·검수 도구 사용법은
[`01-data-status.md`](./01-data-status.md) / [`02-review-guide.md`](./02-review-guide.md) 참조.

> 결정일: 2026-06-22 · 결정: 대표 · 목표: **출시용 핵심대 분석 ON** (해당 대학 모집단위가 `exact`/`parsed`로 분석되는 상태)

---

## 1. 결론 한 줄

기존 핵심대 **50개 프리셋을 유지**하고, 수요/목적별로 **11개 엔트리(실질 9개 기관)를 티어별로 추가**한다. 포항공대는 빼지 않고 플래그로 분리한다.

핵심대 분석 ON은 이미 **약 84% 완료**(기존 50 중 verified 42 / blocker 8). 따라서 5일의 실제 작업은 "핵심대 전수 검수"가 아니라 **① 추가분 검수 + ② blocker는 2027 공개 모니터링으로 분리**다.

---

## 2. 추가 대학 + 검수 우선순위 (티어)

검수는 **must → if_time → eng_special → med_health** 순서로 한다. 모두 2027 단위·rule·evidence가 DB에 있어 바로 검수 가능하다.

| 티어 | 대학 | 2027 단위 | 추가 이유 | 비고 |
|---|---|---|---|---|
| **must (필수)** | 가천대학교 | 13 | 반드시 추가 | 먼저 처리 |
| **if_time (시간되면)** | 경기대학교(서울) | 114 | 정시 수요 | 단위 많음 — 시간 큼 |
| | 경기대학교 | 94 | 정시 수요 | 단위 많음 |
| | 서울여자대학교 | 37 | 정시 수요 | |
| | 동덕여자대학교 | 50 | 정시 수요 | 단위 많음 |
| | 덕성여자대학교 | 1 | 정시 수요 | 2027 단위 희박 |
| **eng_special (공대/특수)** | 한국항공대학교 | 5 | 공대/특수 수요 | |
| | 한국공학대학교 | 5 | 공대/특수 수요 | |
| **med_health (의약/보건)** | 가톨릭대학교(서울) | 49+12+49 | 의약/보건 목적 | **서울 가톨릭대만** (아래 주의) |

### ⚠️ 가톨릭대 주의
DB에서 "가톨릭대"는 9개 기관(대구·광주·대전·인천·부산·수원·목포 등)이 잡힌다. 추가 대상은 **이름이 정확히 "가톨릭대학교"인 서울 본교(성심/성의교정, 의·약·간호)뿐**이다. 지역 가톨릭대는 추가하지 않는다.

### 시간 배분 팁
`if_time` 안에서도 **경기대(208단위)·동덕여대(50)** 는 검수량이 크다. 시간이 빠듯하면 단위 적은 **서울여대 → 덕성여대 → 가천대/항공대/공학대**부터 쳐내고, 경기대·동덕여대는 남는 시간에.

---

## 3. 포항공대 플래그

포항공대는 핵심대에서 **빼지 않는다.** 다만 수시 중심 + 브랜드 대학이라 정시 환산식 신뢰도가 낮아(2027 정시 일반 환산식이 ADIGA raw에 사실상 없음, 현재 blocker), 다음으로 표시한다:

> **"특수/브랜드 대학 · 정시 confidence 낮음"**

프리셋 파일 `flags`에 기록돼 있다. 향후 분석 출력에서 이 대학의 confidence를 보수적으로 강등하는 근거로 쓴다.

---

## 4. blocker (5일 내 완료 불가 — 2027 미공개)

기존 50 중 8개는 2027 공식 산출표 미공개/엔진 범위 밖이라 **노력과 무관하게 지금 verified 불가**다(상세: `02-review-guide.md` §6). 5일 목표에서 제외하고 2027 공개 모니터링(`02-review-guide.md` §7)으로 분리한다:

서강대 · 성균관대 · 중앙대(2엔트리) · 포항공대 · 건국대(일부) · 홍익대(2엔트리)

---

## 5. 작업 분할 (2인)

- 검수는 `/admin/review`에서 한다(localhost 전용, 공유 Supabase에 바로 반영). 사용법: `02-review-guide.md` §3.
- **중복 방지를 위해 대학 단위로 나눈다.** 예) A가 must + eng_special + 가톨릭대, B가 경기대·동덕여대·서울여대·덕성여대. 같은 대학을 둘이 동시에 만지지 않는다.
- 영어 ratio 대학은 웹폼이 아니라 JSONL fill 경로(`02-review-guide.md` §4–5)로 처리한다.
